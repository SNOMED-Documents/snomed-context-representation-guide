# 5.2.2 Refinement Options

# Introduction

In some cases, a single [concept](https://confluence.ihtsdotools.org/display/DOCGLOSS/concept "Glossary link: concept") may not provide sufficient detail to fully represent the required meaning. In these cases, if the [EHR](https://confluence.ihtsdotools.org/display/DOCGLOSS/EHR "Glossary link: EHR") supports the storage of additional refining characteristics these can be used to add specificity to the meaning of a more general _concept_. For example, this approach may be used to specify the site of a particular symptom, injury, or procedure. The [SNOMED CT concept model](https://confluence.ihtsdotools.org/display/DOCGLOSS/SNOMED+CT+concept+model "Glossary link: SNOMED CT concept model")1 specifies the set of [attributes](https://confluence.ihtsdotools.org/display/DOCGLOSS/attribute "Glossary link: attributes") that can be applied to refine the meanings of  _concepts_ in a particular [domain](https://confluence.ihtsdotools.org/display/DOCGLOSS/domain "Glossary link: domain"), and the [ranges](https://confluence.ihtsdotools.org/display/DOCGLOSS/range "Glossary link: ranges") of values that can be applied to each of those  _attributes_.

# Definition

A [refinement](https://confluence.ihtsdotools.org/display/DOCGLOSS/refinement "Glossary link: refinement") is  the part of a [SNOMED CT expression](https://confluence.ihtsdotools.org/display/DOCGLOSS/SNOMED+CT+expression "Glossary link: SNOMED CT expression") that applies qualifying details to a [focus concept](https://confluence.ihtsdotools.org/display/DOCGLOSS/focus+concept "Glossary link: focus concept").

# Representation

As noted above, the _SNOMED CT Concept Model_ specifies the  _attributes_ and _ranges_ that can be used to refine the meaning of concepts in a particular  _domain_ in the definition above, SNOMED CT  _meaning bindings_ are represented as [expression templates](https://confluence.ihtsdotools.org/display/DOCGLOSS/expression+template "Glossary link: expression templates"). 

A [SNOMED CT expression template](https://confluence.ihtsdotools.org/display/DOCGLOSS/SNOMED+CT+expression+template "Glossary link: SNOMED CT expression template") is  a [SNOMED CT template](https://confluence.ihtsdotools.org/display/DOCGLOSS/SNOMED+CT+template "Glossary link: SNOMED CT template") that represents an [expression](https://confluence.ihtsdotools.org/display/DOCGLOSS/expression "Glossary link: expression") in which one or more elements are replaced by [SNOMED CT template slots](https://confluence.ihtsdotools.org/display/DOCGLOSS/SNOMED+CT+template+slot "Glossary link: SNOMED CT template slots"). 

### Notes

  * An _expression template_ includes one or more predefined variables which when replaced by appropriate values creates a conformant _expression_.
  * An _expression template_ can be used to specify how data items entered via several controls in a structured user interface should be combined to represent a composite meaning.
  * _Expression templates_ are also used to formally represent [meaning bindings](https://confluence.ihtsdotools.org/display/DOCGLOSS/meaning+binding "Glossary link: meaning bindings") associated with different [data entry contexts](https://confluence.ihtsdotools.org/display/DOCGLOSS/data+entry+context "Glossary link: data entry contexts").
  * _Expression templates_ are one of several types of templates defined by the SNOMED CT [Template Syntax Specification](https://confluence.ihtsdotools.org/display/DOCSTS/Template+Syntax+Specification). 

As noted in the definition above, SNOMED CT  _refinement_ options are represented as [expression templates](https://confluence.ihtsdotools.org/display/DOCGLOSS/expression+template "Glossary link: expression templates") [Expression Template Language](https://confluence.ihtsdotools.org/display/DOCGLOSS/Expression+Template+Language "Glossary link: Expression Template Language")2.

The general _refinement options_ shown in this guide are illustrative examples. Rather than covering the full range of attributes that are available for the refinement of  _concepts_ in a particular domain, they are limited to providing a few potentially useful examples.

Full details of the refinement options are provided in the following sections of [Appendix A. Concept Model Reference](https://confluence.ihtsdotools.org/display/DOCCCG/Appendix+A.+Concept+Model+Reference):

  * Section [A.1 Domains](https://confluence.ihtsdotools.org/display/DOCCCG/A.1+Domains)
    * Defines each _domain_
    * Specifies the _attributes_ that can be used to refine  _concepts_ in each  _domain._
    * For completeness, also indicates the  _range_ of values that can be applied to each of the  _attributes_ applicable to the domain. However, the  _range_ for each attribute is general and applies to the use of that attribute in any of the domains to which it is applicable. 
  * Section [A.2 Attributes](https://confluence.ihtsdotools.org/display/DOCCCG/A.2+Attributes)
    * Defines each _attribute_
    * Specifies the _range_ of values that can be applied to that _attribute_. The defined  _range_ applies to that  _attribute_ in all  _domains_ to which it is applicable.
