# 5.1.1 Value Set Binding

# Introduction

The values that can rationally be entered in a particular data entry field are logically limited to the set of values that make sense in the context of that field. For example, if the name of a disorder, such as "appendicitis (disease)", is entered in a field with the label "surgical procedure" it would be impossible to determine the intended meaning. Perhaps the author of the record entered this diagnosis in the wrong place on a form or maybe they were attempting to record an "excision of appendix (procedure)" to resolve the patient's appendicitis. This issue applies to free text or coded records. However, where coded records are used, the risks of error may be greater if users search the entire content of a code system for the correct code. A mechanism that limits the results of searches to codes that are appropriate to the specific [data entry context](https://confluence.ihtsdotools.org/display/DOCGLOSS/data+entry+context "Glossary link: data entry context"). The logical mechanism used to address this requirement is referred to as [value set binding](https://confluence.ihtsdotools.org/display/DOCGLOSS/value+set+binding "Glossary link: value set binding"). 

# Definition

A [value set binding](https://confluence.ihtsdotools.org/display/DOCGLOSS/value+set+binding "Glossary link: value set binding") is a [terminology binding](https://confluence.ihtsdotools.org/display/DOCGLOSS/terminology+binding "Glossary link: terminology binding") that represents the set of permitted values that can be used to populate a coded data item.

### Notes

  * A  _value set binding_ can be defined using individual rules. For example concepts that are: in a list of included [concept identifiers](https://confluence.ihtsdotools.org/display/DOCGLOSS/concept+identifier "Glossary link: concept identifiers"), members of a specified [reference set](https://confluence.ihtsdotools.org/display/DOCGLOSS/reference+set "Glossary link: reference set") or [subtypes](https://confluence.ihtsdotools.org/display/DOCGLOSS/subtype "Glossary link: subtypes") of a specified concept. 

  * [Expression constraints](https://confluence.ihtsdotools.org/display/DOCGLOSS/Expression+constraint "Glossary link: Expression constraints") provide a more flexible way to represent _value set bindings_ as they are able to represent and combine a range of different rules to determine the set of [concepts](https://confluence.ihtsdotools.org/display/DOCGLOSS/concept "Glossary link: concepts") or [expressions](https://confluence.ihtsdotools.org/display/DOCGLOSS/expression "Glossary link: expressions") that are part of a _value set_.

# Representation

As noted in the definition above, SNOMED CT [value set bindings](https://confluence.ihtsdotools.org/display/DOCGLOSS/value+set+binding "Glossary link: value set bindings") should be represented as [expression constraints](https://confluence.ihtsdotools.org/display/DOCGLOSS/expression+constraint "Glossary link: expression constraints"). 

A [SNOMED CT expression constraint](https://confluence.ihtsdotools.org/display/DOCGLOSS/SNOMED+CT+expression+constraint "Glossary link: SNOMED CT expression constraint") is a computable rule that is used to define a set of clinical meanings. 

### Notes

  *  _SNOMED CT expression constraints_ are defined using the [expression constraint language (ECL](https://confluence.ihtsdotools.org/display/DOCGLOSS/Expression+Constraint+Language "Glossary link: Expression Constraint Language")).
  * _Expression constraints_ can be used for a variety of purposes including:
    * Specifying [value set bindings](https://confluence.ihtsdotools.org/display/DOCGLOSS/value+set+binding "Glossary link: value set bindings") that constrain the values that can be entered in a particular [data entry context](https://confluence.ihtsdotools.org/display/DOCGLOSS/data+entry+context "Glossary link: data entry context").
    * Representing an [intensional subset definition](https://confluence.ihtsdotools.org/display/DOCGLOSS/intensional+subset+definition "Glossary link: intensional subset definition"), which may be used to determine the membership of a [reference set](https://confluence.ihtsdotools.org/display/DOCGLOSS/reference+set "Glossary link: reference set")
    * Querying [electronic health records](https://confluence.ihtsdotools.org/display/DOCGLOSS/electronic+health+record "Glossary link: electronic health records") for record entries that contain specified sets of [concepts](https://confluence.ihtsdotools.org/display/DOCGLOSS/concept "Glossary link: concepts") and/or [expressions](https://confluence.ihtsdotools.org/display/DOCGLOSS/expression "Glossary link: expressions")
    * Defining the [SNOMED CT concept model](https://confluence.ihtsdotools.org/display/DOCGLOSS/SNOMED+CT+concept+model "Glossary link: SNOMED CT concept model") by specifying the sets of concepts in a [ | concept model domains|](http://snomed.org/fictid# "\(eg:\)  | concept model domains |") and the [range](https://confluence.ihtsdotools.org/display/DOCGLOSS/concept+model+range "Glossary link: concept model range") of values applicable to each [attribute](https://confluence.ihtsdotools.org/display/DOCGLOSS/attribute "Glossary link: attribute").
